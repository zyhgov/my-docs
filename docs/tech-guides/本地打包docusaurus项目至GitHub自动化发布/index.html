本地打包docusaurus项目至GitHub自动化发布

提示

以下是完整的重新打包并强制推送到 gh-pages 分支的步骤，包含详细解释：

1. 本地构建静态文件​

# 进入项目根目录（包含 package.json 的目录）
cd D:\新桌面Desktop\联合库UNHub\UNHUB\Docusaurus\my-docs

# 运行构建命令（生成静态文件到 build/ 目录）
npm run build

  • 作用 ：将 Docusaurus 项目编译为静态 HTML/CSS/JS 文件，输出到 build/ 目录。

  • 关键配置 ： 确保 docusaurus.config.js 中的 baseUrl 为 /my-docs/：

    module.exports = {
      url: 'https://zyhgov.github.io',
      baseUrl: '/my-docs/',  // 必须与仓库名一致
      // 其他配置...
    };

2. 进入构建目录​

cd build
  • 作用 ：切换到生成的静态文件目录，后续操作均在此目录进行。

3. 初始化 Git 仓库​

git init
  • 作用 ：初始化一个新的 Git 仓库（如果目录中已有 .git 文件夹，可跳过此步）。

4. 关联远程仓库​

git remote add origin https://github.com/zyhgov/my-docs.git
  • 作用 ：将本地仓库与 GitHub 远程仓库关联。

5. 切换到 gh-pages 分支​

git checkout -b gh-pages
  • 作用 ：创建并切换到 gh-pages 分支（若分支已存在，直接使用 git checkout gh-pages）。

6. 添加所有文件到暂存区​

git add -A
  • 作用 ：强制添加所有文件（包括新增、修改和删除的文件）。

7. 提交更改​

git commit -m "Deploy to gh-pages"
  • 作用 ：将暂存区的文件提交到本地仓库。

8. 强制推送到远程 gh-pages 分支​

git push origin gh-pages --force
  • 作用 ：覆盖远程 gh-pages 分支的内容，确保本地构建文件完全同步。
  • 注意 ：--force 会删除远程分支的旧提交历史，请谨慎使用。

9. 验证部署​

  1. 访问 GitHub Pages URL ： 打开 https://zyhgov.github.io/my-docs，确认页面正常显示。
  2. 检查控制台 ： 打开浏览器开发者工具（F12），确保无 404 错误。
注意

一些部署上传常见问题处理方法

常见问题处理​

问题 1：404 错误​

  • 原因 ：baseUrl 配置错误或文件未正确上传。

  • 解决

    1. 检查 docusaurus.config.js 中的 baseUrl 是否为 /my-docs/。
    2. 确保 build/ 目录包含 index.html 和静态资源。

问题 2：样式丢失​

  • 解决 ： 检查 build/assets/ 和 build/img/ 目录是否完整上传。

问题 3：缓存问题​

  • 解决 ： 强制刷新浏览器缓存（Ctrl+F5）或使用无痕模式访问。

后续更新流程​

每次修改内容后，重复以下步骤：

npm run build    # 重新构建
cd build         # 进入构建目录
git add -A       # 添加所有文件
git commit -m "Update"  # 提交更改
git push origin gh-pages --force  # 强制推送

通过以上步骤，你的文档将始终同步到 gh-pages 分支，并通过 GitHub Pages 对外提供服务。
